Frontier AI agents require specific architectural components beyond large language models to function effectively. These five essential terms define their capabilities:

    agents.md: A markdown text file placed at the root of a project that acts as an instruction layer. It informs the agent about project-specific rules, such as coding conventions, testing commands, and formatting requirements. Nested versions can exist for sub-projects, with files closer to the working directory taking precedence.
    Agent Skills: Folders containing a skill.md file and necessary resources. These provide modular knowledge that the agent only loads when contextually relevant, preventing unnecessary clutter in the agent's context window.
    MCP (Model Context Protocol): An open protocol that standardizes how agents connect to external tools, databases, and APIs. By using an MCP server, agents can interact with various backends—such as project management or payment systems—without requiring custom connectors for each integration.
    A2A (Agent-to-Agent): An open protocol that enables direct communication and coordination between different AI agents. Agents publish an agent card detailing their function, allowing other agents to delegate specific tasks and hand off workflows efficiently.
    Sub-agents: A pattern where a primary agent spawns child agents to handle specific, isolated pieces of work. Each child agent operates within its own fresh context window and returns results upon completion, allowing for parallel processing and preventing the main agent's context from becoming overloaded.
